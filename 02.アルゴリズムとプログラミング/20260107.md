# データ構造（リスト）

配列とリスト

配列は配列、index でアクセスする、indexがあるので index を使った要素の参照は高速 O(1)、一方で配列への挿入と削除はO（N）になる
リストはポインタがつくので、挿入と削除は早い O(1)、ただし参照はポインタをたどる必要があるのでO(N)になる

* 単方向リスト
* 双方向リスト
* 循環リスト

# スタックとキュー

スタック：LIFO
キュー：FIFO

キューの活用例としてのリングバッファ
再起処理ときたらスタック

# 木構造

2分木の走査アルゴリズム、DFS、BFS

# 探索アルゴリズム

* 線形探索：データの先頭から順に探索する。O（N/2）
* 二分探索：整列されたデータの中央値と目的データを比較して探索する O(log2N)
* ハッシュ探索：ハッシュマップを使う、O(1)

# ソートアルゴリズム

* バブルソート O(n^2)
* 選択ソート O(n^2)
* 挿入ソート O(n^2)、最良のケースで O(n)

* 分割統治法
* クイックソート

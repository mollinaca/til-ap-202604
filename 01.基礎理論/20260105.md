# 数値表現と演算精度

## 固定小数点表示

## 2の補数

(6)10 = (0110)2  
(0110)2 の２の補数 (-6)10 は、

* ビットを反転 0110 → 1001
* 1を足す → 1010

検算  
0110 + 1010 = 10000 → 最上位ビット 1 は捨てられて、 0000 となる

## 浮動小数点表記

Y=±f*(r~e)
f : 仮数
r : 基数
e : 指数

誤差

* 丸め誤差 : 四捨五入（丸め）による精度落ち
* 桁落ち：絶対値がほぼ等しい二つの数の加減算により、有効桁が減る
* 情報落ち：絶対値が大きく異なる二つの数の加減算により、絶対値が小さいほうの値が結果に反映されない
* 打ち切り誤差：無限小数において、一定の桁で打ち切られるために発生する真数との差

# 符号理論

## 誤り訂正・通信路符号化

* ハミング符号：冗長ビットを付加、4ビットのデータに対して3ビットの冗長ビットを付加すれば、2ビットの誤り検出と1ビットの誤り訂正が可能
* CRC：送信ビット列を生成多項式で割ったあまりを検査用としてビット列に付加して送信、受信側では送信側と同じ生成多項式で割り切れるか否かで誤りを判断する、訂正はできない
* パリティチェック：奇遇検査とも呼ばれる、誤り検出用のパリティビットを付加する。奇数パリティ、偶数パリティ、垂直水平パリティがある。垂直水平パリティは誤り検出と訂正ができる
  * 行列において、水平方向ないし垂直方向に、1の数が偶数になるように 0 か 1 を付加する。水平垂直に付加すればどのビットが誤ってるか一位に定まるので、誤り訂正まで可能
